### 🔒 SECURITY & DEPLOYMENT WORKFLOW (REMINDER)
Currently, .env files are tracked in Git for convenience. For a secure production environment, follow these steps later:

1. **Untrack .env files**: 
   - Add `.env*` to `.gitignore`.
   - Run `git rm --cached .env .env.prod backend/.env`.
   
2. **Use .env.example**:
   - Create `.env.example` files containing only the keys (e.g., `DB_PASSWORD=`).
   - Commit these examples so you know which variables the app needs.

3. **VPS Configuration**:
   - On the VPS, manually create the `.env` files or copy from `.env.example`.
   - Fill in the actual production secrets directly on the VPS.
   - This ensures secrets never leak into your GitHub history.
